## 1. 策略概览

### 1.1 策略类型

**均值回归马丁 (Mean-Reversion Martingale)**

利用布林带震荡回归的统计特性,在极端偏离时开仓,通过分层加仓摊薄成本,回归中轨或达到篮子盈利时全部平仓。

### 1.2 交易品种/周期

| 项 | 值 |
|---|---|
| 品种 | XAU/USD (黄金) |
| 周期 | M1 (1分钟) |
| 大势参考 | M30 (可选,默认关闭) |
| 账户类型 | USD账户 (非美分) |
| 杠杆 | 500:1 |
| 合约规格 | 0.01手 = 1盎司 (价格$1变动 = 盈亏$1) |

### 1.3 关键数字

```
最大层数:      11
最大总手数:    0.60
价格覆盖:      $86 (L1 到 L11 累计)
篮子止盈:      $8
回中轨止盈:    $2
强平阈值:      账户50%浮亏
最小账户:      $1000
```

## 2. 核心设计哲学

### 2.1 三层结构

```
① L1 开仓  → 严格信号 + 边界检查
② L2-L11加仓  → 纯距离驱动,无任何过滤
③ 出场   → 篮子TP / 中轨TP / 50%强平
```

### 2.2 设计原则

**"入场谨慎,加仓机械,兜底简单"**

- **入场严**: BB+RSI双信号必须同时满足,加上时段/余额/点差/流动性/M30过滤(可选)
- **加仓纯**: 一旦L1开仓,只看"距离"和"是否亏损",不再受任何信号/趋势/波动率过滤影响
- **兜底硬**: 只有一个真正的止损 — 账户浮亏达50%全平

### 2.3 为什么加仓不用过滤

马丁的数学本质是"越跌越买摊薄成本"。若加仓被过滤器拦截,导致本该摊薄时没摊薄,反而在后续价格回归时无法充分获利。**过滤属于L1择时职责,不属于加仓管理职责**。

## 3. 参数完整参考

### 3.1 信号参数 (L1入场)

| 参数 | 默认 | 说明 |
|------|------|------|
| `Inp_BB_Period` | 30 | 布林带周期(M1) |
| `Inp_BB_Dev` | 2.2 | 布林带偏差(标准差倍数) |
| `Inp_RSI_Period` | 14 | RSI周期 |
| `Inp_RSI_OB` | 72.0 | RSI做空触发阈值 (RSI ≥ 此值可做空) |
| `Inp_RSI_OS` | 28.0 | RSI做多触发阈值 (RSI ≤ 此值可做多) |

### 3.2 M30 大势过滤 (可选)

| 参数 | 默认 | 说明 |
|------|------|------|
| `Inp_M30_Filter` | false | 是否启用M30过滤 |
| `Inp_M30_BB` | 30 | M30 BB周期 |
| `Inp_M30_BBDev` | 2.2 | M30 BB偏差 |
| `Inp_M30_NeutralRatio` | 0.30 | 中性区比例(距中轨<半带宽×此比例视为震荡) |

**逻辑**:
```
价格距M30中轨 < 半带宽×NeutralRatio → 震荡区,双向允许
价格在M30上半区(超出中性区) → M30偏多, 禁做空
价格在M30下半区(超出中性区) → M30偏空, 禁做多
```

### 3.3 加仓参数

| 参数 | 默认 | 说明 |
|------|------|------|
| `Inp_MaxLayers` | 11 | 最大层数 |
| `Inp_LotSeq` | `0.01,0.01,0.02,0.02,0.03,0.04,0.05,0.07,0.10,0.12,0.13` | 11层手数序列 |
| `Inp_DistanceSeq` | `4,5,6,7,8,9,10,11,12,14` | L2-L11 距上层的价格间距(USD) |
| `Inp_MaxTotalLots` | 0.60 | 最大总手数 |

### 3.4 流动性熔断 (仅拦L1)

| 参数 | 默认 | 说明 |
|------|------|------|
| `Inp_Liquidity` | true | 启用流动性熔断 |
| `Inp_LiqRange` | 10.0 | M1单根波动触发阈值(USD) |
| `Inp_LiqPauseSec` | 300 | 熔断后暂停秒数(5分钟) |

### 3.5 出场参数

| 参数 | 默认 | 说明 |
|------|------|------|
| `Inp_TP_Basket` | 8.0 | 篮子止盈(USD) — 达标全平 |
| `Inp_TP_MidMin` | 2.0 | 回中轨最低盈利门槛 |

**回中轨止盈逻辑**: 总盈利 ≥ `Inp_TP_MidMin` AND 价格穿越BB中轨 → 全平

### 3.6 风控

| 参数 | 默认 | 说明 |
|------|------|------|
| `Inp_TotalLoss_Pct` | 50.0 | 总亏损百分比阈值 → 强平 |
| `Inp_MinBalance` | 1000.0 | 最小账户余额,低于此不开新单 |

**唯一开仓后保护**: `(balance - equity) / balance × 100 ≥ Inp_TotalLoss_Pct` 触发全平。

### 3.7 交易参数

| 参数 | 默认 | 说明 |
|------|------|------|
| `Inp_MagicNumber` | 20260622 | 订单魔术号(区分本EA订单) |
| `Inp_MaxSpread` | 80 | 最大点差(点), 0=不限 |
| `Inp_Slippage` | 30 | 滑点容忍(点) |
| `Inp_StartHour` | 3 | 每日开始交易时(服务器时区) |
| `Inp_EndHour` | 22 | 每日停止交易时 |

### 3.8 数据导出

| 参数 | 默认 | 说明 |
|------|------|------|
| `Inp_Export` | true | 启用JSON数据导出 |
| `Inp_ExportMs` | 1000 | 状态文件写入节流(毫秒) |

## 4. 交易逻辑详解

### 4.1 L1 开仓 (`CheckEntry`)

**7道闸门** (按代码执行顺序):

```
① 交易时段: 3-22h                    → 否则等待
② 流动性熔断: 未触发                    → 否则暂停中
③ 账户余额: balance ≥ $1000            → 否则不开
④ 点差: spread ≤ 80点                 → 否则等待
⑤ BB触轨: close[1] 触上轨或下轨          → 否则等待
⑥ RSI极值: RSI[1] ≥ 72 或 ≤ 28         → 否则等待
⑦ M30大势(可选): 允许该方向             → 否则等待
   ↓
执行 trade.Buy/Sell(GetLotForLayer(1))
```

**信号定义**:
```
做空信号 = close[1] ≥ BB_upper[1] AND RSI[1] ≥ 72
做多信号 = close[1] ≤ BB_lower[1] AND RSI[1] ≤ 28
```

### 4.2 L2-L11 加仓 (`CheckMartingaleAdd`)

**4道闸门** (纯马丁,无过滤):

```
① 层数硬限: cycleLayer < 11
② 手数硬限: 累计+下层 ≤ 0.60
③ 必须亏损: CalcTotalProfit() < 0
④ 距离触发: 距上层价格 ≥ Δ_N
   ↓
执行 trade.Buy/Sell(GetLotForLayer(N+1))
```

**距离计算**:
```
lastPrice = 最后一层的开仓价 (不是L1)
currentPrice = 当前 Ask (做多) 或 Bid (做空)
distance = 做多? (lastPrice - currentPrice) : (currentPrice - lastPrice)

Δ_N = DistanceSeq[N-2]   例: L2 需要 Δ_2 = distSeq[0] = $4
```

### 4.3 出场 (`CheckTakeProfit`)

**两种止盈同时监控**:

```
条件A (篮子止盈):
  totalProfit ≥ Inp_TP_Basket (默认$8)
  → 全平, ExportTradeClose("TP_BASKET")

条件B (回中轨止盈):
  totalProfit ≥ Inp_TP_MidMin (默认$2)
  AND 做多时: price ≥ BB_middle[0]
  AND 做空时: price ≤ BB_middle[0]
  → 全平, ExportTradeClose("TP_MIDDLE")
```

### 4.4 强平 (`CheckForceClose`)

```
loss_pct = (balance - equity) / balance × 100
if loss_pct ≥ Inp_TotalLoss_Pct (默认50%):
   全平所有持仓
   ExportTradeClose("总亏强平 X.X%")
```

### 4.5 完整 OnTick 流程

```
OnTick():
   UpdatePeakEquity()

   if 持仓中:
      CheckForceClose()      # 强平检查
      CheckTakeProfit()      # 止盈检查
      if 仍持仓:
         CheckMartingaleAdd()  # 加仓检查

   if 新M1 K线:
      barCount++
      CheckLiquidity()       # 更新流动性状态
      if 空仓:
         CheckEntry()        # 首单信号检查
      UpdatePanel()
      ExportState()
      ExportEquitySnapshot()
```

## 8. 故障排查

### 8.1 EA 挂上后不开仓

**检查步骤**:

1. **交易时段**: 服务器时间是否在 3-22h?
2. **算法交易按钮**: 工具栏是否绿色?
3. **余额**: 是否 ≥ Inp_MinBalance ($1000)?
4. **周末**: 六日休市不交易
5. **实时诊断**: 查看 Experts 日志的 `[诊断]` 消息

**常见拦截原因**:
- `× 未触BB轨` — 信号未出现, 耐心等待 (M1 通常 1-2小时才出一次合规信号)
- `× 触下轨但RSI不够多` — BB触了但RSI没到极值
- `× M30大势拦截` — M30 过滤开启时才有,可关闭 `Inp_M30_Filter`
- `× 余额... < 门槛...` — 检查账户余额

### 8.2 EA 显示旧版本号

**原因**: 源码修改了但未重新编译

**解决**:
```
MetaEditor 打开源文件 → F7 编译 → 从图表删除并重新挂载
```

若仍显示旧版本,可能是标签对象残留:
```
图表右键 → 对象列表 → 删除所有 "BBMartin" 开头的对象
```

### 8.3 JSON 文件未生成

**检查**:
1. `Inp_Export` 是否为 `true`?
2. 路径 `%APPDATA%\MetaQuotes\Terminal\Common\Files\` 是否存在?
3. 是否有权限写入?

### 8.4 面板显示 "流动性暂停"

**原因**: 上一根M1 K线波动 ≥ $10

**行为**: 5分钟内不开 L1(加仓不受影响),自动恢复

**手动跳过**: 若确认市场已恢复,可重新挂载EA

### 8.5 加仓不触发

**必要条件**(全部满足):
- 当前持仓周期存在
- 处于亏损状态
- 距上层价格 ≥ 该层间距
- 未达层数/手数硬限

**常见问题**: 亏损但距离不够 — 需要价格继续逆行到规定距离

## 9. 已知局限

### 9.1 未测试的场景

- L8-L11 深层从未真实触发(实盘数据显示)
- 50% 强平从未触发
- 从未遇到闪崩/停牌/跳空
- 样本量小(17天,85笔)

### 9.2 结构性风险

**回报-风险严重不对称**:
- 单笔最大盈利: $8-10
- 单次强平最大亏损: 账户50%
- 需要极高胜率维持

**无法应对**:
- 黄金单向大趋势(如超过$86单向不回)
- 新闻时段异常波动
- 券商滑点/断线/停牌

### 9.3 参数硬编码

以下不可通过参数调整,需修改代码:
- 交易品种(硬绑 `_Symbol`)
- 只交易一个方向的周期(不支持双向对冲)
- 无法自动复利(首单固定 `LotSeq[0]`)

### 9.4 依赖MT5环境

- 需要 MT5 全程运行且账户在线
- 网络中断可能错过入场/出场
- 无云端备份/远程管理

## 10. 版本历史

| 版本 | 主要变化 |
|------|---------|
| v1.0 | 布林带+RSI+固定网格马丁的原始版 |
| v2.0 | 加入递增距离(5→18) 和 M30 过滤(斜率式) |
| v3.0 | 尝试 "纯信号驱动加仓" — RSI重置机制,实测过严废弃 |
| v4.0 | **距离驱动加仓 + M30过滤(斜率式) + 流动性熔断** |
| v4.1 | 风控简化为单一50%强平 + M30改位置式 + 流动性延长到5min |
| v4.2 | 篮子TP $5→$8 |
| **v5.0** | **加仓移除所有过滤(纯距离) + 加入入场诊断日志** |

### v5.0 详细变化

**删除**:
- 加仓中的 K线间隔限制 (`MinBars`)
- 加仓中的 M1 斜率过滤
- 加仓中的 M1 单边过滤
- 加仓中的流动性检查
- L1 中的 M1 单边/斜率/带宽过滤

**保留**:
- L1 的 BB+RSI 信号
- L1 的时段/余额/点差/流动性/M30过滤(M30可选)
- 加仓的距离触发 + 必须亏损 + 层数手数硬限
- 出场三种方式
- 50% 总亏强平

**新增**:
