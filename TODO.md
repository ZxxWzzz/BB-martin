## 2. 单K线异常移动 → 强平+冻结机制

**依赖**: 任务1完成后,从历史数据反推合理阈值。

**目标**: 在"流动性熔断"之外增加一层更严格的保护——单根M1 K线波动超过某个阈值时,**立即强平所有持仓 + 冻结EA**(不再开新单)。

**与现有机制的区别**:

| 机制 | 阈值 | 行为 |
|------|------|------|
| 流动性熔断(现有) | M1波动 ≥ $10 | 只拦L1, 加仓正常, 5min后恢复 |
| **新黑天鹅熔断** | M1波动 ≥ ?(待定) | **全平+永久冻结**(直到人工解锁) |
| 50%总亏强平 | equity < balance×50% | 全平,周期归零,但可继续 |

**待设计**:
- [ ] 阈值参数 `Inp_BlackSwanRange`: 单根M1 high-low(USD)
  - 建议初值: 根据任务1的历史数据,取过去3年第99百分位
  - 候选范围: $15-25 (覆盖大部分新闻冲击的初震)
- [ ] 触发后行为:
  - [ ] 立即 `CloseAllPositions()`
  - [ ] 设置全局标志 `g_emergencyFrozen = true`
  - [ ] `CheckEntry()` 和 `CheckMartingaleAdd()` 都返回
  - [ ] JSON导出新字段 `emergency_frozen: true`
  - [ ] 面板显示醒目"❗已冻结"
- [ ] 解锁机制:
  - [ ] 选项A: 完全手动(EA重启或手动改参数)
  - [ ] 选项B: N小时后自动恢复
  - [ ] 选项C: 价格回稳后(后续X根K线波动均<Y)自动恢复
  - 推荐: **选项A**(黑天鹅事件需人工评估市场)
- [ ] 日志:
  - [ ] `[黑天鹅] M1波动$X 已超过$Y,全平+冻结!`
  - [ ] 平仓时记录每张订单盈亏,导出到 trades.jsonl 时标记 `close_reason: "BLACK_SWAN"`

**设计原则**:
- 比50%强平更早介入,避免马丁在剧烈行情中被打到极限
- 比流动性暂停更严厉,因为单K线$15+不是普通新闻波动,可能是闪崩/暴涨
- 冻结需要人工评估,防止EA在异常市场中"自动重新交易"导致连续损失

**实现优先级**: 等任务1的统计数据出来再定阈值,代码框架可以先准备。

## 3. 距离序列 vs 手数序列的数学耦合

**问题发现**: 当前两条序列没有严格的数学关联,导致**越深的层加仓反而越难止盈**,与马丁本意完全相反。

**核心公式**:

定义 `lag = 平均成本 - 当前价 = 浮亏 / (累计手数 × 100)`

递推关系:
```
new_lag = (cum_{N-1} / cum_N) × (old_lag + Δ_N)
```

每层达到篮子TP需要的反弹:
```
反弹 = (lag + TP_basket) / (cum × 100)
```

**当前序列的实测反弹要求**:

| 层 | Δ | 累计手数 | lag | 篮子TP反弹 |
|----|---|---------|-----|----------|
| L1 | - | 0.01 | 0 | $8.00 |
| L2 | 4 | 0.02 | 2 | $6.00 ↓ |
| L3 | 5 | 0.04 | 3.5 | $5.50 ↓ |
| L4 | 6 | 0.06 | 6.3 | **$7.67 ↑** |
| L5 | 7 | 0.09 | 8.9 | **$9.78 ↑** |
| L6 | 8 | 0.13 | 11.7 | **$12.31 ↑** |
| L7 | 9 | 0.18 | 14.9 | **$15.39 ↑** |
| L8 | 10 | 0.25 | 18 | **$18.28 ↑** |
| L9 | 11 | 0.35 | 20.7 | **$20.92 ↑** |
| L10 | 12 | 0.47 | 24.3 | **$24.51 ↑** |
| L11 | 14 | 0.60 | 30 | **$30.16 ↑** |

**致命问题**: 从L4起反弹要求一路上升,违背马丁"越深越易止盈"的核心机制。原因是**手数增长速度跟不上距离增长**。

**理想数学关系** (保持lag恒定 = L_target):
```
lot_N = cum_{N-1} × Δ_N / L_target
```

代价: 后期手数指数爆炸,超过 MaxTotalLots 上限。

**三个解决方案待评估**:

**方案A: 距离序列递减** ⭐推荐
```
当前: 4,5,6,7,8,9,10,11,12,14
改为: 14,12,10,9,8,7,6,5,4,3
```
- 优势: 手数序列不动, 仅重排距离
- 逻辑: L2触发门槛高(过滤冲动加仓), 触发后小距离集中摊薄
- 风险: L2迟迟不触发, 错过部分回归窗口

**方案B: 手数指数化, 距离不动**
```
0.01, 0.01, 0.02, 0.03, 0.05, 0.08, 0.12, 0.18, 0.25, 0.35, 0.50
累计 ~1.60手
```
- 问题: 远超 MaxTotalLots 0.60

**方案C: 缩减层数, 重设两条序列**
```
7层
Δ:   3,4,5,7,9,12 (覆盖$40)
手数: 0.01, 0.02, 0.03, 0.05, 0.08, 0.12, 0.20
累计 0.51手
```
- 优势: 覆盖小但每层lag可控
- 劣势: 总覆盖范围缩水

- [x] 数值模拟三个方案,生成"各层反弹要求"对照表 → 详见 [analysis/sequence_simulation.md](analysis/sequence_simulation.md)
- [x] **结论**: 之前手算有误, 基线方案数学正确, 不需要改动。三个替代方案各有缺陷。
