### 问题3: 深层救援效率洼地
- L5-L6 单笔仅 $2-3 (最低)
- 资金效率断崖: L6=0.14 $/(lot·min) 仅为 L1 的 0.4%

## 四、V2B 双引擎设计 (V2A + 引擎B)

### 4.1 引擎A (与V2A完全一致)

引擎A 就是 V2A 的全部逻辑,不变。

### 4.2 引擎B (顺势中轨回归)

**触发条件 (方案Z)**:
```
M30 方向判断 (复用 CheckM30TrendFilter 位置式):
  M30 上半区(超出中性区) → 引擎B 可做多
  M30 下半区(超出中性区) → 引擎B 可做空
  M30 中性区 → 引擎B 不触发

回中轨路径判定:
  过去 N=20 根M1 K线内至少1根触过 BB 外轨
  AND 当前价 从触过外轨的方向 接近中轨
  AND |中轨 - 当前价| < 0.3 × ATR
```

**具体触发逻辑**:

```
做多触发 (M30向上):
  过去20根 M1 至少 1根 close ≤ BB下轨
  AND 当前价 < 中轨
  AND (中轨 - 当前价) < 0.3 × ATR
  → 开多 L1

做空触发 (M30向下):
  过去20根 M1 至少 1根 close ≥ BB上轨
  AND 当前价 > 中轨
  AND (当前价 - 中轨) < 0.3 × ATR
  → 开空 L1
```

### 4.3 引擎B 参数

**手数序列 (V2A减半)**:
```
L1  L2  L3  L4  L5  L6  L7  L8  L9  L10 L11
0.01 0.01 0.02 0.02 0.03 0.03 0.04 0.05 0.06 0.05 0.04
累计 = 0.36手
```

**距离序列 (缩小,顺势本预期反弹小)**:
```
L2-L11: 2, 3, 4, 5, 6, 7, 8, 9, 10, 12 (USD)
累计价格覆盖: $66
```

**反弹序列 (V2A减半)**:
```
L1: 3   L2: 2    L3: 1.5   L4: 1.25
L5: 1   L6: 0.75 L7: 0.5   L8: 0.25
L9: 0.15 L10: 0.1 L11: 0.05
```

### 4.4 引擎B 出场
- **篮子TP**: 达到分层反弹目标即全平
- **强平**: 总亏 ≥ 50% 全平
- **无中轨TP** (自身在中轨附近启动,逻辑矛盾)

### 4.5 引擎B 加仓
- 纯距离驱动 (与引擎A 一致)
- **不再看 M30 后续变化** (加仓时 M30 可能已变,但已开周期保持马丁本质)

### 4.6 冲突处理: A/B 互斥

```
无仓时:
  优先检查引擎A 触发 (BB外轨+RSI极值)
  若A不触发,检查引擎B (回中轨路径 + M30方向)
  两者互斥,不同时开仓

有仓时:
  当前活动引擎(A 或 B)独立管理该周期
  加仓/止盈按该引擎参数执行
  另一引擎完全禁止触发
```

### 4.7 引擎标识追踪

新增全局变量 `activeEngine` (0=空仓, 1=引擎A活动, 2=引擎B活动)

开仓时记录,平仓时清零。JSON 导出附带 `engine` 字段。

## 五、通用参数 (V2A 和 V2B 共享)

### 5.1 交易时段
- 全天交易 (24h)
- **仅周一开盘前 N 小时禁交易** (`Inp_MondaySkipHours = 2`)
  - 避开周一亚洲开盘的低流动性/大点差/跳空时段

### 5.2 风控
- 总亏损 50% 强平
- 最小余额 $1000 (低于此不开新单)

### 5.3 流动性熔断
- M1 单根波动 ≥ $10 触发
- 暂停 300 秒 (5 分钟)
- 仅拦 L1 (加仓不受影响)

### 5.4 交易参数
- 最大点差 80 点
- 滑点容忍 30 点

### 5.5 MagicNumber
- **V2A**: `2026071701`
- **V2B**: `2026071702`
- V1 (生产): 20260622

### 5.6 日志
- 保留 v5 入场诊断日志
- V2B 额外记录引擎标识
