# BB马丁 M1 v4.1 — 真正的距离驱动马丁

## v4.1 vs v4.0 改动

1. **风控合并为单一阈值**：删除回撤15%/浮亏12%/日亏5%三层独立机制，合并为**总亏50%**单一强平阈值
2. **新增最小余额门槛**：账户余额<$1000时**不开新单**（已有持仓继续走完）
3. **M30过滤改为位置式**：不用斜率，改用"价格相对中轨位置 + 中性区"
4. **流动性暂停**：180秒 → **300秒(5分钟)**

---

## 设计哲学（v4 vs v3）

```
v3:  L1靠信号 → L2-L11也靠信号 (本质不是马丁,是多次独立进场)
v4:  L1靠信号 → L2-L11靠距离  (真正的马丁,信号只用于L1择时)
```

**关键认知修正**：马丁的灵魂是"持续摊薄成本"。每一层加仓必须由"价格距离"触发，否则平均成本拉不下来，加仓就是纯放大风险敞口。

## 三、加仓逻辑（v4核心改动）

### 3.1 距离序列（USD）

```
Δ_2  Δ_3  Δ_4  Δ_5  Δ_6  Δ_7  Δ_8  Δ_9  Δ_10  Δ_11
 4    5    6    7    8    9   10   11   12    14
```

**累计覆盖**：4+5+6+7+8+9+10+11+12+14 = **86 USD**

意味着从L1到L11全开需要价格逆行$86，覆盖大部分非黑天鹅波动。

### 3.2 加仓条件清单

```cpp
void CheckMartingaleAdd() {
   if(cycleLayer >= MaxLayers) return;                       // 硬限
   if(barCount - lastAddBar < MinBars) return;               // K线间隔
   if(GetCurrentTotalLots() + nextLots > MaxTotalLots) return; // 手数限
   if(CalcTotalProfit() >= 0) return;                        // 必须亏损

   // 距离触发(核心)
   double lastPrice = GetLastOpenPrice();
   double currentPrice = (cycleDirection==1) ? Ask : Bid;
   double distance = (cycleDirection==1) ? (lastPrice - currentPrice)
                                          : (currentPrice - lastPrice);
   if(distance < GetDistanceForLayer(cycleLayer + 1)) return;

   // 趋势保护(防止顺势加仓加到死)
   if(CheckSlopeFilter(cycleDirection==1)) return;
   if(CheckOuterBandFilter(cycleDirection==1)) return;

   // 流动性保护
   if(IsLiquidityPaused()) return;

   // 全过 → 加仓
   ExecuteAdd();
}
```

### 3.3 没有RSI/BB信号要求的理由

- **距离条件已隐含极值**：价格从L1跌$4-14意味着已经远离BB中轨
- **慢速阴跌不被排斥**：RSI不重置不影响加仓，避免v3孤军奋战的死结
- **趋势保护代替信号过滤**：斜率/单边/流动性过滤负责防黑天鹅，效果更可靠

## 五、M30 大势过滤器（v4.1 位置式）

### 5.1 原理（v4.1 重新设计）

不再用斜率，改用**价格相对中轨位置 + 中性区判定**：

```
M30 BB(30, 2.2) 中轨/上轨/下轨
半带宽 = (上轨 - 下轨) / 2
中性区 = 半带宽 × NeutralRatio (默认0.30)
```

**判断逻辑：**

```cpp
double offset = currentPrice - M30_middle;
double neutral = halfBand * 0.30;

if(|offset| < neutral)        → M30震荡(中性区) → 双向允许
else if(offset > 0)           → 价格在M30上半区 → M30偏多 → 禁做空
else if(offset < 0)           → 价格在M30下半区 → M30偏空 → 禁做多
```

### 5.2 例子

M30: BB上=$5050, 中=$5000, 下=$4950，半带宽=$50，中性区=±$15

| 当前价 | offset | 判定 | 结果 |
|--------|--------|------|------|
| $5005 | +$5 | 中性区(\|5\|<15) | 双向允许 |
| $5025 | +$25 | 上半区(25>15) | M30偏多, 禁做空 |
| $4985 | -$15 | 中性区边缘 | 双向允许 |
| $4975 | -$25 | 下半区(25>15) | M30偏空, 禁做多 |

### 5.3 为什么用位置不用斜率

- **斜率有滞后**：8根K线计算，反应慢，易漏掉急转
- **位置即时反映状态**：当前价相对中轨即定性
- **天然包含震荡判断**：中性区设计避免在M30盘整中误拦
- **更直观**：用户能在图上一眼看出M30是上半区还是下半区

### 5.2 为什么放在L1入场前

- **L1是马丁的起点**：方向选错整个周期都白费
- **加仓不再过滤M30**：一旦L1顺势开了，就让马丁逻辑接管
- **避免M30信号反复让加仓中断**：M30信号变化慢，但变化时打断加仓更危险

### 5.4 数据依赖

只需要 1 个 indicator handle：
```cpp
int bbM30Handle = iBands(_Symbol, PERIOD_M30, 30, 0, 2.2, PRICE_CLOSE);
```

## 六、流动性熔断（v4新增）

### 6.1 触发条件

```
最近1根M1 K线 (high - low) ≥ Inp_LiquidityRange  (默认 $3.0)
```

黄金正常M1波动通常<$1，超过$3意味着新闻闪崩、流动性枯竭、跳空等异常。

### 6.2 行为

```
触发 → 暂停所有交易(L1开仓和加仓) Inp_LiquidityPauseSec 秒 (默认180秒=3分钟)
```

暂停期内：
- 不开新L1
- 不加仓
- 已有持仓继续走止盈/止损/风控逻辑

### 6.3 实现

```cpp
datetime g_pausedUntil = 0;

void CheckLiquidity() {
   double high = iHigh(_Symbol, PERIOD_M1, 1);
   double low  = iLow(_Symbol, PERIOD_M1, 1);
   if(high - low >= Inp_LiquidityRange) {
      g_pausedUntil = TimeCurrent() + Inp_LiquidityPauseSec;
      Print("[流动性] 暂停至 ", TimeToString(g_pausedUntil));
   }
}

bool IsLiquidityPaused() {
   return TimeCurrent() < g_pausedUntil;
}
```

每根新K线开始时调用 `CheckLiquidity()`，每次准备开仓/加仓时调用 `IsLiquidityPaused()`。

## 八、对照场景：慢速阴跌

**v3行为**：
```
L1 @ 5000 RSI=27
价格 5000→4994 (-$6) RSI在25-38盘整
v3: rsiArmed=false 永远不能进 → 不加仓
浮亏继续扩大 → 强平 → 损失大
```

**v4行为**：
```
L1 @ 5000 RSI=27
价格 5000→4996 (-$4) → 满足 Δ_2=$4
其他过滤通过 → 加 L2 (0.01手)
平均成本: (0.01×5000 + 0.01×4996)/0.02 = 4998
继续跌到 4991 (-$5 from 4996) → 满足 Δ_3=$5
加 L3 (0.02手)
平均成本: 4994.5
继续跌或反弹...
```

距离驱动让马丁在阴跌中持续摊薄，反弹小幅就能盈利出场。

## 九、对照场景：M30强趋势下逆势进场

**v3行为**：
```
M30: 中轨持续下降 (强下跌趋势)
M1: 价格触BB下轨 RSI=27
v3: 开多L1 → 顺M1反转,但逆M30大势
价格继续跟随M30大势下跌
L1快速变深亏 → 触发距离 → 加L2 → 继续下跌 → 加L3 → ...
风险: 全部加仓都是逆大势,容易满仓深亏
```

**v4行为**：
```
M30斜率检测: -1.2 USD/8bar, M30 ATR=$0.8, 阈值=$0.4
slope < -0.4 → M30下跌趋势
v4: 拒绝做多 → 不开L1
等待M30震荡或转上行后再考虑
```
